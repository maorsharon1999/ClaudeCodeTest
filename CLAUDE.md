# Claude Code Instructions

## Git & GitHub Workflow

- After completing any meaningful unit of work (new feature, bug fix, refactor, config change), commit and push to GitHub immediately.
- Never let significant progress go uncommitted — if in doubt, commit.
- Use clean, descriptive commit messages that explain the "why", not just the "what".
- Always push to the `main` branch on GitHub after committing so work is never only local.
- This ensures the project can always be reverted to a known good state.
